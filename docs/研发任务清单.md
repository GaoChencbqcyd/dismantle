## Sprint 0：项目初始化（第1周 · 20h）

### 0.1 开发环境搭建
- **工时**：3h
- **描述**：安装 Node.js 22 LTS、Xcode 16+、VS Code；配置 CocoaPods；安装 Expo CLI (`npx create-expo-app@latest dismantle --template blank-typescript`)
- **产出**：Expo 项目可运行在 iOS 模拟器
- **验收**：`npx expo start --ios` 成功启动，显示默认屏幕

### 0.2 Supabase 后端初始化
- **工时**：3h
- **描述**：创建 Supabase 项目，配置数据库 Schema（anxiety_sessions, compass_profiles, user_settings），启用 Auth（email + 未来微信OAuth），配置 Row Level Security
- **产出**：Supabase 项目 URL + anon key，3张核心表就绪
- **验收**：通过 Supabase Dashboard 可查看表结构，RLS 策略已启用

### 0.3 Git 仓库初始化
- **工时**：1h
- **描述**：`git init`，创建 `.gitignore`（node_modules, .expo, ios/Pods, 密钥文件），创建 GitHub 仓库，首次提交
- **产出**：GitHub 仓库就绪
- **依赖**：0.1

### 0.4 CI/CD 配置
- **工时**：2h
- **描述**：配置 GitHub Actions：PR 检查（lint + type check）、合并到 main 时自动构建 Expo、输出 iOS 构建产物
- **产出**：`.github/workflows/ci.yml`
- **依赖**：0.3

### 0.5 环境变量管理
- **工时**：1h
- **描述**：配置 `.env` 文件（SUPABASE_URL, SUPABASE_ANON_KEY），使用 `expo-constants` 管理环境变量，`.env` 加入 `.gitignore`
- **产出**：环境变量配置完成
- **依赖**：0.2

### 0.6 App Icon 与 Splash Screen
- **工时**：3h
- **描述**：设计 App Icon（1024×1024），生成各尺寸图标；配置 Splash Screen 背景色和 Logo
- **产出**：App Icon 全尺寸 + Splash Screen
- **验收**：iOS 模拟器中 App Icon 和启动画面正确显示

### 0.7 基础导航框架
- **工时**：3h
- **描述**：安装 React Navigation，创建底部 Tab Navigator（4个Tab占位），创建 Stack Navigator 骨架，配置深色/浅色主题 Context
- **产出**：可切换 4 个 Tab 的导航框架
- **依赖**：0.1

### 0.8 Apple Developer 账号
- **工时**：4h
- **描述**：注册 Apple Developer Program（$99/年），配置 Bundle ID、生成开发证书和 Provisioning Profile
- **产出**：Apple Developer 账号就绪，可在真机调试
- **备注**：审批可能需要 1-2 天，提前启动

## Sprint 1：设计系统落地（第2-3周 · 25h）

### 1.1 颜色系统与主题
- **工时**：3h
- **描述**：实现 A+B 融合主题系统——日间「理性暖调」（#F5F0EB + #C47B5A）+ 夜间「暗夜微光」（#1A1D24 + #E8A940）。创建 ThemeContext，支持手动切换 + 跟随系统。所有颜色值定义为 TypeScript 常量。
- **产出**：`src/theme/colors.ts`, `src/theme/ThemeContext.tsx`
- **验收**：全局切换主题时所有页面颜色正确过渡

### 1.2 Typography 系统
- **工时**：2h
- **描述**：定义字体层级（caption/body/subtitle/title/heading/display），使用 SF Pro Display（iOS 原生）。创建 Text 组件封装，支持 size/weight/color 预设。
- **产出**：`src/theme/typography.ts`, `src/components/AppText.tsx`
- **依赖**：1.1

### 1.3 Button 组件
- **工时**：2h
- **描述**：Primary（实心强调色）、Secondary（浅色背景）、Ghost（透明+文字色）三种变体。支持 size（sm/md/lg）、loading 状态（spinner）、disabled 状态。触感反馈（`expo-haptics`）。
- **产出**：`src/components/Button.tsx`
- **依赖**：1.1, 1.2

### 1.4 Input 组件
- **工时**：2h
- **描述**：文本输入框封装。支持：placeholder、多行（textarea）、左侧图标、错误状态（红色边框+错误文案）、字符计数。聚焦时边框变为强调色。
- **产出**：`src/components/Input.tsx`
- **依赖**：1.1, 1.2

### 1.5 Card 组件
- **工时**：2h
- **描述**：通用卡片容器。支持：default/interactive（可按压）变体，padding 预设，阴影层级。interactive 模式下 onPress 有缩放反馈动画。
- **产出**：`src/components/Card.tsx`
- **依赖**：1.1

### 1.6 Chip / Tag 组件
- **工时**：1.5h
- **描述**：可选中/不可选中的标签组件。选中态：背景=强调色，文字=白色。用于情绪选择、认知扭曲标签、分类标签。
- **产出**：`src/components/Chip.tsx`
- **依赖**：1.1, 1.2

### 1.7 Toggle Switch 组件
- **工时**：1h
- **描述**：iOS 风格开关。支持 on/off 状态，切换动画。用于提醒开关、设置页面。
- **产出**：`src/components/ToggleSwitch.tsx`
- **依赖**：1.1

### 1.8 Progress Bar 组件
- **工时**：1h
- **描述**：水平进度条。支持 0-100% 进度，动画过渡。用于拆解流程 4 步骤进度指示。
- **产出**：`src/components/ProgressBar.tsx`
- **依赖**：1.1

### 1.9 Toast 组件
- **工时**：1.5h
- **描述**：全局 Toast 通知。底部弹出，自动 2s 消失。支持 success/error/info 变体。使用 Context 全局管理。
- **产出**：`src/components/Toast.tsx`, `src/contexts/ToastContext.tsx`
- **依赖**：1.1

### 1.10 Modal Sheet 组件
- **工时**：2h
- **描述**：底部弹出面板。支持拖拽关闭、点击遮罩关闭。内容区域可滚动。用于选择器、确认操作等场景。
- **产出**：`src/components/ModalSheet.tsx`
- **依赖**：1.1

### 1.11 Empty State 组件
- **工时**：1.5h
- **描述**：空状态占位。图标 + 标题 + 描述 + 可选 CTA 按钮。用于无数据、功能未解锁等场景。
- **产出**：`src/components/EmptyState.tsx`
- **依赖**：1.1, 1.2, 1.3

### 1.12 组件库文档
- **工时**：2h
- **描述**：整理所有组件的使用示例和 Props 文档。不需要 Storybook（独立开发者），在 README 或组件文件头部用 JSDoc 注释即可。
- **产出**：组件 JSDoc 注释完整
- **依赖**：1.1-1.11

## Sprint 2：拆解流程-记录与分类（第4-5周 · 30h）

### 2.1 首页-今日状态
- **工时**：3h
- **描述**：实现首页。今日拆解状态环形图（灰色=未完成/绿色=已完成），「开始拆解」大按钮（未完成时），「再做一次」按钮（已完成时）。底部展示最近 3 条拆解记录摘要。
- **产出**：`src/screens/HomeScreen.tsx`
- **验收**：按钮文案随完成状态变化；最近记录正确展示
- **依赖**：1.3, 1.5

### 2.2 首页-情绪快捷选择
- **工时**：2h
- **描述**：横向滚动的情绪卡片（焦虑/烦躁/迷茫/压力/失眠/其他），每张卡片：emoji + 标签。点击选中后展开拆解入口。
- **产出**：`src/components/EmotionPicker.tsx`
- **依赖**：1.5, 1.6

### 2.3 拆解流程-导航容器
- **工时**：3h
- **描述**：创建拆解流程的 Stack Navigator。管理步骤间导航（记录→分类→重构→行动→总结）。实现中途退出时的状态保留和恢复提示。
- **产出**：`src/navigation/DismantleStack.tsx`
- **依赖**：0.7

### 2.4 记录页-文字录入
- **工时**：3h
- **描述**：多行文本输入框（placeholder：「描述让你焦虑的事情...」），字符计数（无上限但建议 500 字内）。底部焦虑强度评分（1-10 横向数字按钮，选中高亮）。「下一步」按钮（未输入文字时 disabled）。
- **产出**：`src/screens/RecordScreen.tsx`
- **验收**：输入文字后按钮启用；评分可选中
- **依赖**：1.4, 1.3, 1.6

### 2.5 记录页-语音录入
- **工时**：4h
- **描述**：使用 `expo-speech-recognition` 或 iOS Speech framework 实现语音录入。录音按钮（按住说话），波形动画（`react-native-reanimated`），最长 120 秒自动截断。转录文本可编辑后确认。权限处理：无麦克风权限时隐藏语音按钮，引导去设置。
- **产出**：`src/components/VoiceRecorder.tsx`
- **验收**：录音→转录→可编辑→确认流程完整；权限拒绝时正确降级
- **依赖**：2.4
- **风险**：中文语音识别准确率。缓解：提供文字输入保底

### 2.6 分类页-因素解析
- **工时**：2h
- **描述**：将用户输入的文本按句号/分号/换行拆分为焦虑因素列表。如果只有一个因素则直接展示。支持手动编辑拆分结果。
- **产出**：`src/utils/parseFactors.ts`
- **依赖**：2.4

### 2.7 分类页-可控/不可控拖拽
- **工时**：5h
- **描述**：核心交互。焦虑因素以卡片形式展示在中间「待分类」区域。上方「✅ 我可控」区域 + 下方「🌊 不可控」区域。卡片支持拖拽（`react-native-gesture-handler` + `react-native-reanimated`）。拖入区域时触感反馈（`expo-haptics`）。全部分类完成后「下一步」按钮启用。也支持点击切换（移动端备选交互）。
- **产出**：`src/screens/ClassifyScreen.tsx`, `src/components/DragZone.tsx`, `src/components/DragItem.tsx`
- **验收**：拖拽流畅（≥60fps）；释放时有触感；点击切换可用；全部分类后按钮启用
- **依赖**：2.6, 1.8

### 2.8 分类页-引导提示
- **工时**：1h
- **描述**：首次使用时展示引导提示：「把焦虑因素拖到对应区域。把精力放在你能改变的事情上。」仅在首次展示，之后不再出现。
- **产出**：分类页内联提示
- **依赖**：2.7

### 2.9 会话状态管理
- **工时**：3h
- **描述**：创建 AnxietySession 的 Zustand Store。管理会话生命周期：idle → recording → classifying → reframing → acting → completed。支持中途退出后恢复（从 localStorage/AsyncStorage 读取未完成会话）。
- **产出**：`src/stores/sessionStore.ts`
- **依赖**：无

### 2.10 本地数据持久化
- **工时**：2h
- **描述**：使用 `expo-secure-store`（敏感数据）+ `@react-native-async-storage/async-storage`（普通数据）实现本地持久化。会话数据、用户设置、罗盘结果本地存储。
- **产出**：`src/utils/storage.ts`
- **依赖**：0.1

## Sprint 3：拆解流程-重构与行动（第6-7周 · 30h）

### 3.1 重构页-认知卡片展示
- **工时**：4h
- **描述**：横向 FlatList 滑动浏览 10 种认知扭曲卡片。每张卡片：编号、名称、一句话描述、自查问题。卡片宽度 = 屏幕宽度 × 0.7，左右露出部分下一张卡片（snap 效果）。选中时卡片展开显示反驳引导文案 + 文本输入框。
- **产出**：`src/screens/ReframeScreen.tsx`, `src/components/DistortionCard.tsx`
- **验收**：10 张卡片完整可滑动；选中展开/收起动画流畅
- **依赖**：1.5, 2.9

### 3.2 认知扭曲卡片内容
- **工时**：2h
- **描述**：将 PRD 中定义的 10 种认知扭曲内容（名称、描述、自查问题、反驳引导）写入数据文件。支持后续从远程配置更新。
- **产出**：`src/data/distortions.ts`
- **验收**：10 条数据完整，文案准确

### 3.3 重构页-反驳输入
- **工时**：2h
- **描述**：选中认知卡片后，展示文本输入框供用户写下反驳内容。输入后保存到会话状态。可跳过（标记为「已浏览未重构」）。未选择任何卡片时也可进入下一步。
- **产出**：重构页反驳输入区域
- **依赖**：3.1, 2.9

### 3.4 行动页-智能推荐
- **工时**：3h
- **描述**：基于焦虑文本的关键词匹配，推荐 3 个微小行动建议。匹配逻辑：文本包含「工作/汇报/老板」→ 工作类；「钱/财务/存款/房」→ 财务类；「催婚/关系/朋友」→ 关系类；「身体/熬夜/健康」→ 健康类；「意义/空虚/迷茫」→ 意义感类。匹配不到则展示默认 3 条。
- **产出**：`src/utils/actionRecommender.ts`
- **依赖**：3.5

### 3.5 微小行动内容库
- **工时**：2h
- **描述**：将 PRD 中定义的 30 条微小行动建议（5 大类 × 6 条 + 默认 3 条）写入数据文件。每条包含：图标、标题、提示文字。
- **产出**：`src/data/microActions.ts`
- **验收**：30+ 条数据完整

### 3.6 行动页-卡片选择
- **工时**：2h
- **描述**：3 张行动建议卡片纵向排列。每张卡片：图标 + 标题 + 提示文字 + 右侧圆形勾选框。点击选中（单选），勾选框填充强调色+对勾。选中后「完成拆解」按钮启用。
- **产出**：`src/screens/ActionScreen.tsx`, `src/components/ActionCard.tsx`
- **验收**：单选逻辑正确；选中动画流畅
- **依赖**：3.4, 1.5

### 3.7 总结页-评分对比
- **工时**：3h
- **描述**：完成动效（🎉/😌 emoji 弹入）。拆解前/后焦虑评分对比（大数字 + 箭头 + 下降分数）。选中的微小行动展示。两个按钮：「返回首页」+「分享」（生成卡片图片）。
- **产出**：`src/screens/SummaryScreen.tsx`
- **验收**：评分对比正确显示；返回首页回到 Tab 主页
- **依赖**：2.9, 3.6

### 3.8 分享卡片生成
- **工时**：3h
- **描述**：使用 `react-native-view-shot` 截取总结卡片生成图片，调用系统 Share Sheet 分享。分享内容不包含具体焦虑文本（隐私保护），仅包含评分变化和日期。
- **产出**：`src/components/ShareCard.tsx`
- **依赖**：3.7

### 3.9 拆解流程-端到端测试
- **工时**：3h
- **描述**：手动走查完整的 4 步流程：文字录入 → 语音录入 → 拖拽分类 → 认知卡片选择+反驳 → 行动选择 → 总结。测试中途退出恢复、空输入处理、超时处理。
- **产出**：测试记录
- **依赖**：2.3-3.8

### 3.10 拆解流程-性能优化
- **工时**：2h
- **描述**：确保拖拽帧率 ≥ 60fps；FlatList 卡片滑动不卡顿；页面切换动画流畅。使用 React DevTools Profiler 检测性能瓶颈。
- **产出**：性能优化后的代码
- **依赖**：3.9

## Sprint 4：人生罗盘（第8周 · 25h）

### 4.1 罗盘主页
- **工时**：2h
- **描述**：功能入口卡片列表：价值观排序（显示 Top 5 摘要或未完成状态）、满意度评估（显示完成状态）、方向分析（完成前两者后解锁）、小步实验。未完成项目显示引导文案，已完成显示 ✅。
- **产出**：`src/screens/CompassScreen.tsx`
- **依赖**：1.5

### 4.2 价值观卡牌-选择页
- **工时**：4h
- **描述**：24 张价值观卡牌以 3 列网格展示。每张卡片：价值观名称。点击选中（最多 5 张），选中后卡片变为强调色背景+白色文字。超过 5 张时 Toast 提示。选中 5 张后「确认」按钮启用。已选中的可再次点击取消。
- **产出**：`src/screens/ValuesScreen.tsx`, `src/components/ValueCard.tsx`
- **验收**：24 张全部展示；选中/取消逻辑正确；最多 5 张限制生效
- **依赖**：1.5, 1.6

### 4.3 价值观卡牌-排序页
- **工时**：3h
- **描述**：选中 5 张后进入排序页。5 张卡片纵向排列，支持拖拽排序（`react-native-draggable-flatlist`）。排序结果保存到 Compass Store。
- **产出**：排序页（可合并到 ValuesScreen 或独立页面）
- **验收**：拖拽排序流畅；顺序正确保存
- **依赖**：4.2

### 4.4 满意度雷达图
- **工时**：5h
- **描述**：使用 `react-native-skia` 绘制 6 维度雷达图。6 个滑杆（工作/财务/关系/健康/成长/生活，1-10 分）。滑动时雷达图实时更新（多边形顶点随分数变化）。保存按钮。
- **产出**：`src/screens/RadarScreen.tsx`, `src/components/RadarChart.tsx`
- **验收**：雷达图正确绘制 6 轴；滑杆拖动时多边形实时更新；分数正确保存
- **依赖**：1.1
- **风险**：Skia 在低端设备上性能。缓解：降低 Canvas 分辨率；备选 victory-native

### 4.5 方向分析页
- **工时**：3h
- **描述**：价值观 Top 5 vs 满意度分数对比列表。识别差距 ≥ 3 分的维度，标记「差距大」。生成关键发现文案（基于最大差距的维度）。底部「创建小步实验」按钮。
- **产出**：`src/screens/AnalysisScreen.tsx`
- **验收**：对比列表正确；差距识别正确；文案合理
- **依赖**：4.3, 4.4
